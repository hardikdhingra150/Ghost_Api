# Week 13 deployment plan

GhostAPI should deploy first on Render.

Render is the best first production target because GhostAPI is a long-running Node/Fastify server that also needs Playwright browser automation. Serverless platforms are not the right first home for this shape.

## Production target

```txt
Browser extension
  → Render HTTPS API
  → GhostAPI Fastify server
  → Playwright workflow runner
  → SQLite now, Supabase Postgres later
```

## Services

| Service | Use | Status |
| --- | --- | --- |
| Render | Hosts the API, dashboard, static assets, and Playwright runtime | selected |
| Supabase Postgres | Production database for users, workflows, runs, and API keys | planned |
| Upstash Redis | Queue/cache layer for cloud workers and rate limits | planned |

No real cloud credentials are committed in this repository.

## Render setup

1. Push this repo to GitHub.
2. Open Render and create a new Blueprint from `render.yaml`, or create a Docker web service manually.
3. Set the service root to the project folder if the repo contains extra parent folders.
4. Keep these environment variables:

   ```txt
   HOST=0.0.0.0
   PORT=4000
   GHOSTAPI_MODE=cloud
   GHOSTAPI_DEPLOYMENT_PROVIDER=render
   GHOSTAPI_REQUIRE_API_KEY=true
   GHOSTAPI_PUBLIC_API_URL=https://your-render-service.onrender.com
   GHOSTAPI_API_KEY=<generated by Render>
   ```

5. Deploy.
6. Open:

   ```txt
   https://your-render-service.onrender.com/health
   https://your-render-service.onrender.com/dashboard
   https://your-render-service.onrender.com/v1/deployment/plan
   ```

## Local production checks

Run these before deploying:

```bash
npm run check
npm run test:deployment
npm run test:extension
npm run package:extension
```

## Cost note

This repo is prepared for Render's free-style starting path, but production browser automation can need paid compute as usage grows. Supabase and Upstash also have free-entry tiers in many cases, but production reliability should be budgeted as a paid system.
